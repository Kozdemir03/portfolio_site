Font-Specific OCR

Lightweight CNN-based character recognition system

Python OpenCV PyTorch ONNX Runtime

Motivation

General-purpose OCR engines (Tesseract, EasyOCR) are large, slow, and overengineered for documents where the font is known in advance — such as internal reports, invoices, or scanned books. This project takes a narrower approach: train a small CNN specifically on a handful of common fonts and deploy it with a fraction of the resource cost.

Target Fonts

  • Times New Roman
  • Arial
  • Georgia
  • (extensible to additional fonts by regenerating the synthetic dataset)

Processing Pipeline

  • Pre-processing: OpenCV converts the input image to greyscale, applies adaptive thresholding, and runs contour detection to locate character bounding boxes.
  • Segmentation: bounding boxes are filtered (area thresholds) to remove noise, then sorted left-to-right and top-to-bottom for correct reading order.
  • Classification: each 32 × 32 cropped bounding box is passed through a MobileNetV3-small (or custom lightweight CNN) trained on the synthetic dataset.
  • Reconstruction: classified characters are concatenated by their sorted position to produce the final text string.
  • Export: the trained PyTorch model is converted to ONNX for fast CPU inference with ONNX Runtime — no GPU required at inference time.

Synthetic Dataset

Training data is generated entirely with Python + Pillow: characters are rendered at varied sizes, rotations, and colours on randomly-noised backgrounds. The target is 50–100K samples, split across the three font families and the full printable ASCII character set. This avoids the need to source and license real scanned documents.
